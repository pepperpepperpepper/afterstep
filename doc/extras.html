AfterStep Wharf/MonitorWharf extras (optional)

AfterStep’s NeXTStep-style edge “dock” is provided by the Wharf and MonitorWharf modules. Many buttons are opportunistic: if the referenced application is not installed (not in $PATH), the button may be disabled/hidden. This is why a fresh install can look “sparse”.

Tip: the default Wharf config includes a “Wharf extras” button that opens this page.

Modern defaults

  • Terminal: configured via ExecInTerm so it can use any terminal emulator.
  • Common apps: Wharf tries modern, widely-used applications first (Firefox/Chromium, Thunar/Dolphin/Nautilus, etc.).
  • Fallback monitors: if classic dockapps aren’t present, MonitorWharf falls back to built-in tools like top, df, and ip via watch inside a terminal.

To override the terminal AfterStep uses, set $TERMINAL (recommended), or edit TermCommand in ~/.afterstep/base.

Arch Linux (examples)

Install a modern “everyday” set from the official repos (populate the docks with a terminal, browser, file manager, editor, and mixer): 

sudo pacman -S --needed xterm xdg-utils firefox thunar mousepad pavucontrol mc htop

For the classic dockapp look, many dockapps are in the AUR. Examples that match the default Wharf/MonitorWharf config: 

yay -S --needed asclock-classic wmcpuload wmnet wmnd wmtop wmfire wmitime wmclockmon wmmixer alsamixer.app wmusic ascd

If a package name above doesn’t exist (AUR names sometimes vary), search: 

yay -Ss asclock
yay -Ss wmcpuload

What maps to what

Dock button Modern equivalents (repos) Classic dockapps (often AUR)
Terminal xterm, alacritty, kitty, xfce4-terminal —
Clock xclock asclock, wmclockmon, wmitime
CPU / load top, htop wmcpuload, wmtop, wmfire
Disk df asfsm, asfsm_lite
Network ip (iproute2) wmnet, wmnd
Mixer pavucontrol, alsamixer wmmixer, AlsaMixer.app, asmixer
Browser xdg-open + your default browser —
File manager thunar, dolphin, nautilus, nemo, pcmanfm —
TUI file manager mc, ranger, nnn, lf —
Mail xdg-email + your default mail handler asmail
IRC / chat hexchat, weechat, irssi BitchX

Customize

Copy the system Wharf config to your home config and edit it: 

cp /usr/share/afterstep/wharf ~/.afterstep/wharf

Then restart AfterStep (or use the “Restart” menu item) to reload the config.
